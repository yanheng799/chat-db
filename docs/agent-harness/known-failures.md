## 失败分类说明

- **一次性失败**：特定环境下偶发，不需要系统性修复
- **环境问题**：依赖服务不可用、网络问题、配置缺失
- **测试不稳定**：测试本身的问题（flaky test），非代码缺陷
- **真实缺陷**：代码逻辑错误，需要修复

## 记录触发条件

遇到以下情况时必须记录：

1. 构建或安装失败
2. 测试意外失败（非 TDD Red 阶段）
3. 服务启动失败
4. Lint/类型检查报错且原因不直观
5. 外部服务连接失败
6. agent 执行任务时卡住且原因与项目配置或结构相关

重复出现的失败应转入 `harness-debt.md` 并建议进入技术债链路。
