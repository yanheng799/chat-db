一、系统概述

1.1 项目目标

构建一个基于多 Agent 协作的自然语言数据库查询系统，让用户能够通过对话式交互，使用日常口语甚至行业黑话查询数据库。系统自动理解意图、标准化查询值、规划查询步骤、生成并执行只读 SQL，关键节点支持可配置的人工审核。系统具备错误自愈能力，能够自动处理元数据过期等问题。

1.2 核心特性

· 零注释容忍：即使数据库表字段无注释，也能自动推断语义
· 语义匹配：支持口语化、行业黑话输入，四层递进自动映射
· 查询值标准化：自动处理时间、枚举、区域、名称简称
· 多 Agent 协作：联合分析、计划生成、执行审核分工协作
· 单步/多步自适应：简单查询直接执行，复杂查询自动拆解
· 可配置审核：根据查询复杂度、数据量、语义来源决定是否审核
· 错误自愈：元数据过期自动同步、SQL 错误自动修复
· 用户特征学习：记录查询偏好、技能水平，持续优化体验
· 多轮对话：支持上下文记忆、结果缓存复用

1.3 不做的事情（V1 明确边界）

· 不支持多数据源（V1 单数据源）
· 不自动量化模糊数值（追问用户）
· 不支持财年/农历等需计算的自定义周期
· 不建独立的复合值逻辑解析器（交由 LLM 处理）
· 不对枚举值做向量化（用确定性规则）
· 不做性能预算控制（保留监控埋点）
· 不做名称词典定时同步（用 fallback 机制）

五、错误自愈系统

5.1 核心认知

```
表名/字段名错误 99% 来自元数据过期 → 重点在同步机制
语义匹配偏差不会触发数据库报错 → 需主动检测
每次错误都是学习机会 → 反馈到元数据和匹配模型
```

5.2 自愈架构

```
SQL 执行失败
    │
    ▼
错误分类
├── 表不存在 → 实时刷新表列表 → 结构相似度匹配 → 自动更新/告知
├── 字段不存在 → 刷新字段列表 → 名相似度匹配 → 自动更新/查找关联表
├── SQL语法错误 → LLM重写 → 语法校验 → 重试
├── 类型不匹配 → LLM添加转换函数 → 重试
└── 其他 → 告知用户 + 记录日志
```

5.3 关联表字段自愈

```python
def handle_cross_table_field(missing_column, current_table):
    # 1. 向量库全局搜索
    matches = vector_db.search('field_descriptions', missing_column, top_k=10)
    
    # 2. 过滤关联表
    candidates = [m for m in matches if m.table != current_table and m.score > 0.7]
    
    # 3. 查图谱找 JOIN 路径
    for candidate in candidates:
        path = graph_db.find_shortest_path(current_table, candidate['table'])
        if path:
            return auto_join_result(candidate, path)
    
    # 4. 无法 JOIN → 返回候选
    return candidate_list_result(candidates)
```

6.1 同步策略

```python
class MetadataSyncEngine:
    """
    两种同步触发方式：
    1. 定时巡检：每天凌晨全量扫描
    2. 错误触发：SQL 执行失败时立即同步相关表
    """
    
    def scheduled_sync(self):
        """定时全量同步"""
        diff = compute_schema_diff(cached_schema, current_schema)
        if diff.has_changes:
            apply_diff(diff)
            notify_admin(diff)
    
    def error_triggered_sync(self, table_name):
        """错误触发同步：仅刷新出错的那张表"""
        current = fetch_current_table_schema(table_name)
        cached = get_cached_table_schema(table_name)
        
        if current != cached:
            update_table_metadata(table_name, current)
            return detect_change_type(cached, current)
```

十五、关键设计决策

决策点 选择 理由
Agent 编排 LangGraph，合并澄清+分析+值标准化 减少串行 LLM 调用
语义匹配 四层递进（词典→行业→向量→LLM） 确定性优先、成本可控
业务指标 词典锁定，禁止 LLM 动态编造 保证核心业务准确性
