# 项目上下文

## 术语表

| 术语 | 定义 |
|------|------|
| 目标数据源 | 用户通过系统查询的业务数据库（PG 或 MySQL）。连接信息通过前端管理界面配置，存储在应用数据库中。 |
| 应用数据库 | 系统自身使用的 PostgreSQL 实例，存储元数据、用户画像、审计日志、值映射、数据源配置等。 |
| 元数据 | 从目标数据源提取的 schema 信息，包括表、字段、索引、外键等结构描述。按 `data_source_id` 隔离。 |
| 数据源配置 | 目标数据源的连接信息（引擎类型、host、port、用户名、加密密码、数据库名、schema 白名单等），存于应用数据库。 |
| 同步 | 比较目标数据源当前 schema 与已存储的元数据，检测差异并更新。 |
| 激活数据源 | 多条数据源配置中当前被激活的那一条。系统查询和元数据提取针对激活数据源。 |

## 角色

| 角色 | 描述 |
|------|------|
| 管理员 | 配置数据源、管理元数据、配置审核策略。通过管理端界面操作。 |
| 查询用户 | 使用自然语言查询数据库的业务人员。通过 Chat UI 操作。 |

## 通用规则

- 数据源密码使用对称加密存储，加密密钥通过 `.env` 的 `ENCRYPTION_KEY` 配置
- API 返回数据源配置时不包含密码明文
- 系统始终只针对激活数据源执行查询和元数据操作
- `.env` 仅存应用基础设施配置（应用 PG、Redis、Neo4j、加密密钥等），目标数据源连接信息不入 `.env`
