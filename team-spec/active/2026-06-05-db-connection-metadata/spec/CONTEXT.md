# 数据源连接与元数据提取 — 需求上下文

## 需求范围术语

| 术语 | 定义 |
|------|------|
| 只读连接 | 对目标数据源建立的连接，强制只读权限，用于元数据提取和后续 SQL 执行 |
| 全量提取 | 首次连接目标数据源时，提取指定 schema 的所有表、字段、索引、外键信息 |
| 全量同步 | 定时扫描目标数据源，对比已存储元数据，检测并记录结构变更 |
| Schema 白名单 | 数据源配置中指定的 schema 列表（PG）或 database 列表（MySQL），未指定则提取所有非系统 schema |

## 本需求涉及的包

| 包 | 职责 |
|------|------|
| `src/config/` | 应用配置（.env → pydantic-settings），含加密密钥 |
| `src/db/` | 目标数据源连接管理（只读、连接池、超时） |
| `src/metadata/` | 元数据提取、存储、同步引擎 |
| `src/api/` | 数据源配置 CRUD API、连接测试 API |

## 系统边界

- Phase 1 的连接层仅用于元数据提取，不执行用户查询 SQL（Phase 5）
- Phase 1 的同步引擎仅做定时全量同步，不做错误触发同步（Phase 7）
- Phase 1 不同步向量库和图谱（Phase 3 建立后才需要）
- Phase 1 不提取行数
